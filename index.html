Cárdenas

Witness the vastness of what you've actually accomplished

The Name

Named after García López de Cárdenas, the first European to see the Grand Canyon in 1540. He saw the scale but couldn't comprehend it fully—which perfectly captures the feeling of looking at your own activity data.

Like Cárdenas standing at the rim, the system helps you witness the vastness of what you've actually accomplished, even when you can't fully grasp it in the moment.

What It Does

Cárdenas follows the principle of conversational simplicity: just talk about what you're doing throughout the day, and the system captures everything in a queryable format.

  • Zero-friction logging to daily JSON files
  • Intelligent querying by date range
  • Goals with horizons (life → yearly → quarterly → monthly → weekly)
  • Health and mood tracking
  • Local-first, no external dependencies

Philosophy

Capture depth, not just summaries. Good entries include mental state, strategic insights, and the "why" behind decisions.

BAD: "Wrote document"
GOOD: "Mental state: Using work momentum as self-soothing—creating control when system feels chaotic"

Get Started

Copy this prompt and paste it into Claude Code. It will set up the complete system for you. 

Set up Cárdenas, a personal activity tracker for conversational logging.

Create these scripts:
1. `track` - logs timestamped entries to `activity/raw/daily/YYYY-MM-DD.json`
2. `scripts/read-activity` - queries activities by --today, --week, --days N, --since, --range
3. `scripts/goals` - manage goals with horizons: life, yearly, quarterly, monthly, weekly
4. `scripts/goal-reflect` - log progress against goals
5. `scripts/health-log` - log mood (stress, energy) and symptoms

Directory structure:
- activity/raw/daily/ (JSON files)
- activity/summaries/daily/ (narrative summaries)
- goals/active.json, goals/reflections/
- health/daily/

Philosophy: Capture DEPTH, not just summaries. Good entries include:
- Mental state and emotional context
- Strategic insights and breakthroughs
- Why certain approaches are being chosen
- Direct quotes capturing key thinking

Example: BAD: "Wrote document" → GOOD: "Mental state: Using work momentum as self-soothing - creating control when system feels chaotic"

Data format: JSON arrays with {time, activity} objects.

Make it local-first, no external dependencies. Just fast local files.
